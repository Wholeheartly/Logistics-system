# 跨境电商订单核算与物流优化助手 — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位

面向跨境电商卖家的智能助手，基于预置的产品数据库、物流费率表和邮编分区表，提供两大核心能力：**订单批量核算与对账**、**多渠道路由比价与最便宜发货方案推荐**。全程离线运行，不调用外部 API。

### 1.2 目标用户

- 跨境电商卖家（主要使用场景：美国市场发货）
- 运营/财务人员（订单对账场景）
- 供应链/物流专员（发货渠道选择场景）

### 1.3 核心价值

- 一键批量核对订单应收与实收差异，快速定位异常订单
- 基于 SKU 和目的地邮编，自动计算五大物流渠道的预估总费用，推荐最经济的发货方案
- 将所有计费规则（Zone 分区、附加费、燃油费率等）内置于系统，用户无需手动查表

## 2. 功能需求

### 2.1 订单批量核算与对账

#### 2.1.1 输入

用户以表格粘贴或键值对列表形式提供订单数据，每条订单至少包含：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 订单号 | 唯一标识 | 必填 |
| SKU | 产品 SKU | 必填 |
| 商品总价 | 商品售价总和 | 必填 |
| 运费 | 向买家收取的运费 | 必填 |
| 税 | 税费 | 按场景 |
| 折扣 | 订单折扣金额 | 按场景 |
| 平台补贴 | 平台补贴/优惠 | 按场景 |
| 买家支付金额 | 买家实际支付总额 | 必填 |
| 平台费用 | 平台扣除的手续费等 | 必填 |

**缺字段处理：** 当输入缺少计算所需的必填字段时，主动列出缺失项并引导用户补充，不强行猜测。

#### 2.1.2 计算规则

```
应收金额 = 商品总价 + 运费 + 税 - 折扣 - 平台补贴
实收金额 = 买家支付金额 - 平台费用
差异金额 = 应收金额 - 实收金额
```

#### 2.1.3 判定逻辑

| 条件 | 状态 |
|------|------|
| 差异 = 0 | 正常 |
| 差异 ≠ 0 | 异常 |

#### 2.1.4 输出

Markdown 表格，包含以下列：

| 订单号 | SKU | 应收金额 | 实收金额 | 差异金额 | 状态 |
|--------|-----|---------|---------|---------|------|

异常行可附简短原因说明。

### 2.2 物流费用查询与最便宜方案推荐

#### 2.2.1 输入

| 字段 | 说明 | 是否必填 | 默认值 |
|------|------|---------|-------|
| SKU | 产品 SKU（支持多个，可分开报价或合并包裹） | 必填 | - |
| 目的地邮编 | 收件地址邮编 | 必填 | - |
| 发货仓库 | CA / NJ / TX / SAV | 选填 | CA |
| 是否住宅地址 | 住宅地址触发住宅附加费 | 选填 | 否 |

#### 2.2.2 处理流程

1. 根据 SKU 从产品数据库获取尺寸（长/宽/高 cm）和毛重（kg）
2. 单位转换：kg → lbs（× 2.20462），cm → inch（÷ 2.54）
3. 根据目的地邮编前三位和发货仓，查分区表确定 Zone
4. 分别对五个渠道并行计算预估总费用
5. 按总费用升序排列，标注最便宜的 1-2 个方案
6. 支持多个 SKU 场景：
   - 分开报价（每个 SKU 独立计算）
   - 若尺寸重量允许，判断是否可合并为一件包裹计算

#### 2.2.3 通用计费公式

```
计费重(lbs) = max(实重, 体积重)
体积重 = 长(inch) × 宽(inch) × 高(inch) / 体积除数
总费用 = 基础运费 + 最高附加费 + 住宅附加费(如适用) + 偏远附加费(如适用) + (基础运费 + 最高附加费) × 燃油费率
```

**附加费互斥规则：** AHS-Dim（超长）、AHS-Weight（超重）、AHS-Packaging（异形）、Oversize（超大）四项仅收取其中最高的一项，不重复收取。

**单位换算常数：**
- 1 kg = 2.20462 lbs
- 1 inch = 2.54 cm

## 4. 邮编分区逻辑

### 4.1 发货仓选项

| 仓代码 | 所在区域 |
|--------|---------|
| CA | 西部 |
| NJ | 东部 |
| TX | 南部 |
| SAV | 东南部 |

### 4.2 分区规则

- 根据目的地邮编**前三位**和所选发货仓，从对应分区表中查找 Zone（2-8）
- 用户未指定发货仓时，**默认使用 CA 仓**，并在结果中提示
- 极偏远邮编（如 AK、HI、PR 等），部分渠道可能无法配送，需提醒用户
- 超出预置分区表的邮编，提示无法判断并建议用户核实

## 5. 备用功能：CG 价卡 3.1.1 US Pick & Ship Standard

仅在用户明确询问 CG 仓发货时启用，不在物流比价主流程内。

### 5.1 计费公式

```
总费用 = Base Rate + $/LB × (实重 - 1 lb)
实重向上取整
```

### 5.2 费率表

| 类别 | Base Rate | $/LB | 触发条件 |
|------|-----------|------|---------|
| Small | $17.05 | $0.20 | — |
| Medium | $23.78 | $0.24 | Small 体积 > 10,368 in³ |
| Large | $49.83 | $0.31 | Medium 体积 > 17,280 in³ |
| Oversize | $52.43 | $0.34 | — |

### 5.3 限制条件

- Small 体积超 10,368 in³ → 按 Medium 计
- Medium 体积超 17,280 in³ → 按 Large 计
- 超限不可用 Multichannel：最长边 > 108 inch，或 长+围长 > 165 inch，或实重 > 150 lbs

## 7. 输出规范

### 7.1 对账结果输出

Markdown 表格，列定义：

| 列名 | 说明 |
|------|------|
| 订单号 | 用户提供的订单标识 |
| SKU | 产品 SKU |
| 应收金额 | 计算出的应收 |
| 实收金额 | 计算出的实收 |
| 差异金额 | 应收 - 实收 |
| 状态 | 正常（差异=0）/ 异常（差异≠0） |

### 7.2 物流推荐输出

Markdown 表格，列定义：

| 列名 | 说明 |
|------|------|
| 物流渠道 | 渠道名称 |
| 计费重量(lbs) | 最终用于计费的重量 |
| 区号(Zone) | 查表得到的 Zone |
| 基础运费 | 查表得到的基础运费 |
| 最高附加费 | 四项附加费中取最高 |
| 住宅附加费 | 如适用 |
| 偏远附加费 | 如适用 |
| 燃油费 | (基础运费 + 最高附加费) × 燃油费率 |
| 总费用 | 以上各项之和 |
| 时效参考 | 预估送达时间 |
| 备注 | 如"最便宜"标签 |

### 7.3 通用要求

- 所有金额保留**两位小数**
- 货币单位与用户输入一致，默认为 **USD**
- 计算结果末尾必须附加提示：「以上运费为估算值，以实际下单为准」

## 8. 约束与限制

### 8.1 功能边界

- 所有运费为**模拟估算**，不代替物流商实时报价
- **不执行任何真实交易操作**
- **不访问外部 API**，仅基于内部预置数据和规则回复
- 涉及金额、税率等敏感信息时，提示用户咨询专业人员

### 8.2 已知局限

- 燃油费率默认为固定估算值（20%），非实时数据
- 邮编分区表覆盖范围有限，超出范围无法判断
- 部分渠道（如 FDX Ground Economy）计费重上限为 35 lbs，超限不可用
- CG Pick & Ship 为备用功能，不在主流程中自动启用
