## 二、详细审查发现

### 2.1 费用计算核心逻辑（shipping_calculator.py）

**审查结果：逻辑正确，无漏洞**

计算步骤：
1. 单位转换（cm→inch, kg→lb）
2. 查 Zone（warehouse + zip_prefix）
3. 计算计费重 = max(实重, 体积重)，向上取整
4. 检查重量范围（min_weight_lb ~ max_weight_lb）
5. 查基础运费表
6. 计算附加费（evaluate_surcharges）
7. 计算偏远附加费
8. 计算住宅附加费
9. 计算 DAR + Non-Mach
10. **燃油费 = (基础运费 + 最高附加费) × 燃油费率**
11. **总价 = 基础运费 + 最高附加费 + 住宅费 + 偏远费 + DAR + Non-Mach + 燃油费**
12. **不含燃油费总价 = 基础运费 + 最高附加费 + 住宅费 + 偏远费 + DAR + Non-Mach**

**边界条件处理：**
- 除零保护：dim_factor 有默认值 139.0
- 空值保护：fuel_rate 使用 `getattr(carrier, 'fuel_rate', 0.0) or 0.0`
- 重量超限检查：计费重超过 carrier.max_weight_lb 返回 None

### 2.2 附加费计算逻辑（surcharge_service.py）

**审查结果：逻辑正确，互斥规则正确实现**

互斥附加费类型：
- AHS-Dim
- AHS-Weight
- AHS-Packaging
- Oversize
- AHS-NonStandard

**取最高逻辑：**
```python
mutually_exclusive = ["AHS-Dim", "AHS-Weight", "AHS-Packaging", "Oversize", "AHS-NonStandard"]
highest_amount = 0.0
for st in mutually_exclusive:
    amt = details.get(st, 0.0)
    if amt > highest_amount:
        highest_amount = amt
```

**AHS-Packaging 默认费用：**
- 当前实现：`details["AHS-Packaging"] = get_surcharge_amount(...)` 无条件查询
- 数据库默认值：根据 seed_database.py 导入数据，AHS-Packaging 默认值为 0.0 或已调整为零
- **结论：符合要求，无需修改**

### 2.3 对账引擎（reconciliation_engine.py）

## 四、新增功能说明

### 4.1 物流区域比价功能

**API 端点：** `POST /api/shipping/zone-compare`

**功能：**
- 支持同时选择多个发货仓进行比价
- 返回各仓库的 Zone 编号和运费详情
- 提供全仓库综合排名（按总价升序）
- 标记最便宜的 1-2 个选项

**前端界面：**
- 新增"单仓比价/区域比价"模式切换
- 多选发货仓（CA/NJ/TX/SAV）
- 综合排名表显示发货仓信息

### 4.2 发货区域配置模块

**API 端点：**
- `GET /api/zones` - 查询区域配置列表
- `POST /api/zones` - 新增区域配置
- `POST /api/zones/{id}/update` - 更新 Zone 编号
- `POST /api/zones/{id}/delete` - 删除区域配置
- `GET /api/zones/warehouses` - 获取仓库列表

**前端界面：**
- 配置管理页面新增"发货区域配置"入口
- 支持按仓库和邮编前缀筛选
- 支持分页（20/50/100/200/500 条/页）
- 弹窗表单新增/编辑
- 删除确认对话框

### 4.3 不含燃油费的总价字段

**后端字段：** `total_without_fuel`

**显示位置：**
- 单仓比价结果表
- 区域比价综合排名表
- 各仓库详情表

**计算逻辑：**
```
total_without_fuel = base_freight + highest_surcharge + residential_fee + remote_fee + dar_fee + non_mach_fee
```

## 八、建议

1. **AHS-Packaging 触发条件优化**：当前 AHS-Packaging 费用无条件查询，建议后续增加用户界面标记非标准包装的功能，以便准确触发该附加费。

2. **燃油费率动态更新**：当前燃油费率为固定值，建议考虑从外部API定期更新燃油费率。

3. **区域比价性能**：当仓库和渠道数量增加时，建议考虑并行计算各仓库运费以提升响应速度。

4. **发货区域配置导入导出**：建议后续增加Excel批量导入/导出功能，方便管理员批量维护区域配置。
